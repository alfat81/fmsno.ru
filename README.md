# fmsno.ru — сайт Федерации мотоциклетного спорта Нижегородской области

Статичный сайт на чистом HTML/CSS/JS. Деплоится на **GitHub Pages**.

## Структура

```
fmsno.ru/
├── index.html             # Главная
├── competitions.html      # Соревнования (календарь, CRUD через админку)
├── documents.html         # Нормативные документы
├── disciplines.html       # Дисциплины
├── news.html              # Новости (с импортом из ВК)
├── partners.html          # Партнёрам (форма заявки)
├── contacts.html          # Контакты (форма обратной связи)
├── style.css              # Все стили
├── env.js                 # ТОКЕНЫ (см. ниже)
├── telegram.js            # Telegram-бот: отправка новостей/заявок
├── js/
│   ├── app.js             # Sticky header, мобильное меню
│   ├── notifications.js   # Общая функция showNotification
│   ├── events-manager.js  # Админка + соревнования + новости
│   ├── documents-manager.js
│   ├── feedback.js        # Заявки в админку
│   └── vk-import.js       # Импорт постов из ВК
├── images/                # Логотипы, фото
├── .nojekyll              # Запрет Jekyll на GitHub Pages
├── CNAME                  # fmsno.ru
├── robots.txt
└── sitemap.xml
```

## Доступ к админке

1. В footer любой страницы нажмите «Вход для администратора»
2. Логин: `admin`
3. Пароль: `fmsno2025`

В админке доступны:
- Добавление / редактирование / удаление соревнований
- Добавление / редактирование / удаление новостей
- Импорт новостей из ВКонтакте (кнопка «Импорт из ВК»)
- Просмотр заявок из форм «Контакты» и «Партнёрам» (кнопка «Заявки» с badge)

⚠️ **ВАЖНО про безопасность**: пароль и токены видны в исходниках браузера.
Это норма для статичного сайта. Не храните так что-то ценное. Для реальной
защиты нужен мини-бэкенд (Cloudflare Worker / Vercel Function / и т.п.).

## Деплой на GitHub Pages

### Вариант 1: custom domain (fmsno.ru)

1. Создайте репозиторий (например `fmsno.ru` или `fmsno-website`)
2. Залейте в корень все файлы из этого архива
3. Settings → Pages → Source: `main` branch, `/ (root)` folder
4. В Settings → Pages → Custom domain: впишите `fmsno.ru`
5. У файла `CNAME` уже прописан `fmsno.ru` — он должен лежать в корне
6. На стороне регистратора домена добавьте запись:
   ```
   A     @     185.199.108.153
   A     @     185.199.109.153
   A     @     185.199.110.153
   A     @     185.199.111.153
   CNAME www   username.github.io.
   ```
7. Дождитесь активации HTTPS (Settings → Pages → Enforce HTTPS)

### Вариант 2: репозиторий `username.github.io`

Если репозиторий называется `<ваш-логин>.github.io`, сайт будет доступен по
адресу `https://<ваш-логин>.github.io/` и `CNAME` **не нужен** (удалите его).

### Вариант 3: project page `username.github.io/reponame`

Если репозиторий называется иначе, сайт будет по адресу
`https://<ваш-логин>.github.io/<reponame>/`. В этом случае:
- Удалите `CNAME`
- Все ссылки в HTML относительные (`index.html`, `competitions.html` и т.д.)
  — это уже так, поэтому работать должно.

## Настройка интеграций

### Telegram-бот (для авто-постинга и уведомлений о заявках)

Токен уже в `env.js`. Чтобы заменить на своего бота:

1. Создайте бота через [@BotFather](https://t.me/BotFather)
2. Получите токен и ID канала (через [@getidsbot](https://t.me/getidsbot))
3. Впишите в `env.js`:
   ```js
   TELEGRAM_BOT_TOKEN: 'ваш_токен',
   TELEGRAM_CHAT_ID:   '-100xxxxxxxxxx',
   ```
4. Добавьте бота администратором в ваш Telegram-канал

### VK API (для автозагрузки новостей из группы vk.com/fmsno)

1. Зайдите на [dev.vk.com](https://dev.vk.com/)
2. Создайте Standalone-приложение
3. В настройках найдите «Сервисный токен»
4. Впишите его в `env.js`:
   ```js
   VK_SERVICE_TOKEN: 'ваш_сервисный_токен',
   ```
5. Без этого токена автозагрузка ленты ВК недоступна (работает только
   ручной импорт по ссылке в админке)

## Ограничения GitHub Pages

- ✅ Статичные файлы любого размера до 1 GB
- ✅ Кастомный домен с HTTPS
- ✅ 100 GB трафика в месяц
- ❌ **Нет бэкенда** — все «секреты» в `env.js` видны всем
- ❌ **Нет серверной логики** — все данные в `localStorage` браузера
- ⚠️ Файлы с `_` в начале имени игнорируются Jekyll, но у нас есть `.nojekyll`

## Хранение данных

Все соревнования, новости, документы и заявки хранятся в `localStorage`
браузера пользователя. Это значит:

- У каждого посетителя свой набор данных (для гостей только seed-данные)
- У админа свои изменения видны только в его браузере
- Очистка кэша браузера → данные сбрасываются к seed-значениям

Это компромисс для статичного сайта без бэкенда. Если нужно, чтобы
изменения были видны всем — нужен бэкенд с базой данных.
